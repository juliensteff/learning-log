## Notes

- **CRLF problem.** Windows saves `\r\n`, Linux/Mac use `\n`. Git treats them as different bytes, so every line of a re-saved file shows as "modified" with no real content change. `.gitattributes` ships the fix with the repo so it applies to anyone who clones.
- **index.lock.** Git creates it during operations to prevent two processes corrupting the index. If a git process crashes — or VS Code's source-control panel hangs on it — the lock stays. `rm .git/index.lock` is the cure.
- **Notes are non-negotiable for retention.** Skim-watching the cert without notes left nothing. Notes-first restart of Module 1 actually stuck.
- **Comment discipline.** File-level header explains *why* the file exists. Sparse inline comments only for non-obvious choices. Code that needs a comment on every line is code that needs better names. Real codebases keep them sparse.
- **PEP 8 spacing.** No space between function name and `()`. Caught the habit early: `def main ()` → `def main()`.
- **Tutor scope.** Working tool, private. The artifacts (code, notes, decisions) are public. Mention the tutor practice in the README — don't push the tutor's own files.
